# CLAUDE.md — project context for Claude Code

Native macOS **menu-bar app** that runs, supervises and backs up the OHF
matter.js-based **`matter-server`** (npm package, the successor to the Python
Matter Server). The app embeds a Node.js runtime + `matter-server` so end users
need nothing pre-installed. Public repo: `nicx/matter-server`.

## Build & run

```bash
./Scripts/bundle-runtime.sh   # downloads Node (host arch) + npm-installs matter-server into ./Runtime, then slims it
./Scripts/make-app.sh         # swift build -c release + assembles & ad-hoc-signs dist/MatterServer.app
open dist/MatterServer.app
```

`swift build` alone compiles the executable (Xcode/Swift 6 toolchain, macOS 14+).
`Runtime/` and `dist/` are git-ignored and regenerated by the scripts.
`UNIVERSAL=1 ./Scripts/bundle-runtime.sh` builds a universal (arm64+x86_64) Node.

### Signing gotcha
The embedded `node` binary **must** be signed with the JIT entitlements
(`allow-jit`, `allow-unsigned-executable-memory`) — otherwise the hardened
runtime kills V8 at startup ("Failed to reserve virtual memory for CodeRange").
`make-app.sh` handles this; keep it if you change signing.

## Architecture (Sources/MatterServer/)

- `MatterMenuBarApp.swift` — `@main` app, `AppDelegate`, `AppEnvironment` (wires
  all managers + the server-down email handler), MenuBarExtra scene, Settings,
  Logs window. Monochrome mesh icon (`point.3.connected.trianglepath.dotted`).
- `ServerController.swift` — Node `matter-server` child process lifecycle. Guards
  against duplicate processes (guard on process handle, not status), keepalive
  with backoff, `terminateNow()` on quit, `onOutage` callback after sustained
  failure, orphan reclaim before every launch (see below).
- `BundledRuntime.swift` — resolves bundled Node + server entry
  (`dist/esm/MatterServer.js`, **not** a bin), builds CLI args, reads
  `installedServerVersion`/`matterSdkVersion` from package.json.
- `BackupManager.swift` — atomic `.zip` snapshots via `ditto` (write `.partial`
  then move), retention, restore, daily timer.
- `UpdateChecker.swift` — daily npm `latest` check; emails once per new version.
- `Mailer.swift` — sends via `curl smtp://<relay>` (plain, no auth/TLS locally).
- `AppSettings.swift` — UserDefaults-backed config (`com.matterserver.app`).
- `Views/` — MenuContentView, SettingsView (Server/Backup/General tabs), LogView.

## Key facts & decisions

- matter-server entry is `dist/esm/MatterServer.js`; CLI flags it parses include
  `--storage-path`, `--port` (5580), `--primary-interface`, `--bluetooth-adapter`
  (no plain `--ble`), `--log-level`, `--enable-test-net-dcl`.
- **Versions are independent**: `matter-server` (the server, e.g. 1.0.0), the
  matter.js SDK `@matter/*` (e.g. 0.17.2), and this app — all shown separately.
- **An orphan is detected by the port, not by a lock file.** A hard app crash
  skips `terminateNow()` and leaves a node process holding port 5580; without a
  handle every keepalive spawns a doomed instance (exit 1) — an endless loop only
  a reboot cleared in the sibling apps. `ServerController.reclaimOrphanServer`
  runs before each launch: it probes the port by binding it (`SO_REUSEADDR`, IPv4
  — that catches Node's dual-stack `::` socket, while a `::` probe would *miss* an
  IPv4-only one), asks `lsof` for the holder, and signals only if `ps` shows our
  install + storage path. matter.js's own `matter.lock` is no help here: it
  self-heals via a PID liveness check, so it never marks a live orphan.
- Default paths: storage `~/Library/Application Support/MatterServer/storage`,
  backups `…/MatterServer/backups` (configurable), logs `~/Library/Logs/MatterServer/`.
- App size ~169 MB: ~114 MB Node + matter.js server. `bundle-runtime.sh` strips
  build-only dashboard libs (vis-network/@mdi/@material) — they're inlined in the
  dashboard's served `dist/web` bundle, so safe to drop.
- `--enable-test-net-dcl` (Settings → Advanced) lets non-CSA test/dev devices
  (e.g. DIY Tasmota) pass device attestation.
- Known limitation: real device commissioning may need the macOS
  `com.apple.developer.networking.multicast` entitlement (Apple-granted;
  documented/commented in `Resources/MatterServer.entitlements`).

## Working agreements (carried over from prior sessions)

- **Commit & push automatically once a change is safe for this public repo**
  (no secrets/keys, no unintended files, `Runtime/`/`dist/`/`.build/` stay
  git-ignored, diff matches intent). End commit messages with the Co-Authored-By
  trailer. If anything looks unsafe/ambiguous, stop and ask.
- The user runs a **local SMTP relay (MailRelay)** at `127.0.0.1:2525`
  (`~/Git/mailrelay`): submit plaintext SMTP there, no auth/TLS on the local hop;
  it forwards upstream. The update/outage emails use it.
