# MatterServer

A native macOS **menu bar app** that runs, supervises and backs up the
[OHF Matter Server](https://github.com/matter-js/matterjs-server) (the
matter.js-based `matter-server`, the successor to the Python Matter Server).

The app is a thin native supervisor around the server's Node.js process:

- **Monochrome menu-bar icon** (SF Symbol template — adapts to light/dark and reflects status).
- **Start / Stop / Restart** with crash auto-restart (keepalive, exponential backoff).
- **Open Dashboard** (`http://localhost:<port>`) and a live **log viewer**.
- **Backups**: timestamped `.zip` snapshots of the storage directory, configurable
  destination folder, daily schedule, retention (keep last *N*), restore, and
  optional "stop server during backup" for a consistent snapshot.
- **Server settings** in-app: port, storage path, `--primary-interface`,
  `--bluetooth-adapter`, `--log-level`; plus an npm-registry update check.
- **Start at login** via `SMAppService`.

Self-contained: a Node.js runtime and `matter-server` are embedded in the
`.app`, so end users need nothing pre-installed.

## Requirements

- macOS 14 (Sonoma) or newer.
- Xcode 16+ / Swift 6 toolchain to build.
- Network access at build time (to download Node.js and `matter-server`).

## Build & run

```bash
# 1. Fetch the embedded runtime (Node.js + matter-server) into ./Runtime
./Scripts/bundle-runtime.sh

# 2. Build and assemble the .app bundle (ad-hoc signed for local use)
./Scripts/make-app.sh

# 3. Launch — the icon appears in the menu bar
open dist/MatterServer.app
```

The bundle is ~215 MB and host-architecture only (`arm64` on Apple Silicon).
For a universal binary that also runs on Intel, run
`UNIVERSAL=1 ./Scripts/bundle-runtime.sh` before step 2.

For signed distribution, set `CODESIGN_IDENTITY="Developer ID Application: …"`
before step 2, then notarize the resulting bundle.

> **Note on signing:** the embedded `node` binary is signed *with* the JIT
> entitlements (`allow-jit`, `allow-unsigned-executable-memory`). Without them
> the hardened runtime blocks V8 and Node aborts at startup. `make-app.sh`
> handles this; keep it in mind if you change the signing steps.

### Developing without bundling

You can iterate on the Swift UI/logic without building the full bundle:

```bash
./Scripts/bundle-runtime.sh   # prepare ./Runtime once
swift run                     # uses ./Runtime as a dev fallback
```

Override paths during development with `MATTER_RUNTIME_DIR`, `MATTER_NODE_PATH`
or `MATTER_SERVER_DIR`.

## Default locations

| What | Path |
|---|---|
| Server storage | `~/Library/Application Support/MatterServer/storage` |
| Backups | `~/Library/Application Support/MatterServer/backups` (configurable) |
| Logs | `~/Library/Logs/MatterServer/matter-server.log` |

## Project layout

```
Sources/MatterServer/
  MatterMenuBarApp.swift   App entry, AppDelegate, scenes, menu-bar icon
  AppSettings.swift        UserDefaults-backed configuration
  ServerController.swift   Node process lifecycle + keepalive + log capture
  BundledRuntime.swift     Resolves bundled Node + matter-server entry, builds args
  BackupManager.swift      ditto snapshots, retention, restore, daily timer
  LoginItemManager.swift   SMAppService start-at-login
  LogStore.swift           In-memory ring buffer + log file
  Views/                   MenuContentView, SettingsView, LogView
Resources/                 Info.plist, MatterServer.entitlements
Scripts/                   bundle-runtime.sh, make-app.sh
```

`Runtime/` (the embedded Node.js + matter-server) and `dist/` (the built app)
are generated by the scripts and are git-ignored.

## Known limitations

- **Multicast/mDNS**: Matter commissioning uses link-local multicast. On macOS
  the `com.apple.developer.networking.multicast` entitlement may be required
  (granted by Apple via the Developer portal). It is documented but commented
  out in `Resources/MatterServer.entitlements`. Non-sandboxed apps usually work
  without it.
- Bluetooth commissioning uses an HCI adapter id (`--bluetooth-adapter`), a
  Linux/D-Bus concept that is typically unavailable on macOS.
- Running the matter.js server outside its supported container is considered
  experimental upstream; Thread-based devices in particular may be unreliable.
- Bundle size is ~215 MB (host arch) due to the embedded Node.js runtime — by
  design, for a zero-dependency install.
